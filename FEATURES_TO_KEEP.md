# FEATURES TO KEEP

## 1. 모바일 뷰어에서 반드시 유지할 기능
- 교사별 시간표를 휴대폰에서 볼 수 있어야 한다.
- 오늘 요일 기준으로 오늘 시간표를 빠르게 확인할 수 있어야 한다.
- 학사일정을 함께 볼 수 있어야 한다.
- 자동 로그인 또는 사용자 식별 기능이 최대한 유지되어야 한다.
- 화면이 모바일에서 읽기 쉽게 표시되어야 한다.
- 데이터 오류가 있어도 앱 전체가 바로 죽지 않도록 해야 한다.

## 2. 데스크톱 앱에서 반드시 유지할 기능
- 교사별 시간표를 PC에서 볼 수 있어야 한다.
- 학사일정을 함께 확인할 수 있어야 한다.
- 트레이 실행 또는 빠른 실행 방식은 가능한 유지 검토한다.
- 로고와 아이콘이 정상 표시되어야 한다.
- 시간표와 학사일정 데이터를 쉽게 교체할 수 있어야 한다.
- GitHub에서 최신 데이터 업데이트를 받을 수 있어야 한다.
- 데이터 오류가 있을 때 기존 데이터를 유지할 수 있어야 한다.

## 3. 공통으로 반드시 지켜야 할 원칙
- 모바일과 데스크톱은 같은 데이터 구조를 사용한다.
- 시간표 파일명과 학사일정 파일명은 고정한다.
- GitHub data 폴더를 기준으로 업데이트한다.
- 업데이트 전에는 새 데이터를 검증한다.
- 검증 실패 시 기존 데이터를 덮어쓰지 않는다.
- 현재 운영 중인 기존 저장소는 계속 분리 유지한다.

## 4. 첫 차세대 버전에서 욕심내지 않을 기능
- 복잡한 관리자 편집 기능
- 화려한 테마 추가
- 부가 통계 기능
- 대규모 UI 개편
- 여러 형식의 데이터 파일 동시 지원
